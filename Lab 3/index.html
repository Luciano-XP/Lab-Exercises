Server

The server's official name is Gucaserver. The origin to the name is that it uses the the first four letters of an old username and I liked how it sounded for the server name and will be referencing it as such. Gucaserver is running Proxmox and uses a mix of LXC and Docker containers in/alongside virtual machines. This server in particular has gone through many full-wipe revisions, the current version being the 4th in the process. There are ways to streamline backup processes and whatnot, but I like the zen that comes from setting up the server from the ground up each time (I do backups of important files though, of course). There is also a second server that I have in the cloud hosted by Linode. This server uses a simple WireGuard VPN in tandem with nifty IPTables configurations to push ports 80 and 443 (HTTP & HTTPS) from Gucaserver, to the WireGuard server, and then to the internet to be accessed by the world. The purpose of this is to bypass the need for port forwarding and making myself and the server that much more secure.

Revision 5 of Gucaserver will include a second server with improved specs to create a Proxmox cluster with (although I am considering learning kubernetes for this revision). Not only that, I have three single board computers that I want to integrate into the main server as well. These SBCs currently run software that I do not want to have direct connection to the main server with such as password managers and pastebins.

The next version of the server will also have Docker-Compose files for each of the services hosted on GitHub for easy access for myself and others.
